# Refactor Code

## Usage
기존 코드를 가독성, 유지보수성, 성능 관점에서 개선한다.

## Checklist
- [ ] 중복 코드 제거 (DRY)
- [ ] 함수/컴포넌트 단일 책임 원칙 적용
- [ ] 타입 정의 명확화 (any 제거)
- [ ] 불필요한 useEffect 제거
- [ ] 서버 컴포넌트로 전환 가능 여부 검토
- [ ] 에러 핸들링 일관성 확인
- [ ] 성능: 메모이제이션, 코드 스플리팅 검토

## Constraints
- 기능 변경 없이 구조만 개선
- 리팩토링 전후 동작 동일성 확인
- 과도한 추상화 지양
